Introdução

Introdução

  • Solução ORM para a plataforma .NET
  • Estrutura para mapear um modelo OO para um BD Relacional
  • Fork do Hibernate
  • Open Source
  • Distribuído sob a licença GNU

História

História

  • O projeto teve inicio por Tom Barrett em meados de 2003
  • No final de 2005, JBoss deixa de apoiar o projeto
  • Com isso o projeto passou a ser 100% liderado pela comunidade

Melhores práticas

  • Fechar as sessões que não estão sendo utilizadas

Vantagens

Vantagens

  • Produtividade
  • Baixo acoplamento
  • Facilita a manutenção do código
  • Evita injeções de SQL
  • Muito utilizado, facilitando o suporte caso assim necessário for
  • Cache 2nd level

Desvantagens

Desvantagens

  • Não é nativo
  • Depende de XML para configuração
  • Dependendo do projeto, sua performance não justifica a sua utilização

Conclusão

Conclusão

Seguindo o sucesso do Hibernate, o fork NHibernate para o .NET é considerado uma das melhores soluções para a plataforma .NET/C#. Atualmente, com sua comunidade de desenvolvimento reduzida e a concorrência do Entity Framework (agora também open source), o futuro do NHibernate é incerto. Apesar disso, ainda é uma solução de respeito, e que deve ser levada em consideração na hora da escolha de um framework ORM para a plataforma .NET/C#.
